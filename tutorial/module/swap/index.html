MetaBase Swap use overview

MetaBase Swap is an innovative and extremely fast token exchange protocol. It uses the contract layer transaction method for token exchange, so the transaction method is very different from the traditional DEX.

Swap transaction

Exchange of tokens in the MetaBase platform is a very easy and simple thing.

Enter the Swap page, in the upper left corner of the transaction area, select the transaction currency, enter the transaction quantity, and select "Buy" or "Sell".

Swap assets automatically complete the transfer in the contract, without authorization and other operations, the transaction fee is low, and the operation is simple.

Swap Note
  1. The corresponding assets in the MetaBase contract need to be recharged first, otherwise it cannot be traded.
  2. When the transaction pair does not exist, MetaBase will give priority to the automatic routing function to indirectly complete the target transaction.
  3. When Route is not possible, trading may not be possible, and trading pairs need to be adjusted or liquidity added.
  4. The final amount obtained by Swap is subject to the actual completion of the transaction, and the page data is for reference only.

Liquidity Management

Any user can become an MetaBase LP market maker to add liquidity to the trading pair to obtain IMM market-making income. Market makers may obtain relevant participation rights and discounts in the platform's marketing activities. The market making operation is very simple. In the Swap trading area, click "Add Liquidity" or "Remove Liquidity". When market makers remove liquidity, the incremental income they obtain is market-making income and commission dividends.

Transaction mining

In order to encourage users to participate in project transactions, the platform also provides a new incentive tool for project parties-"transaction mining". Any user can provide transaction mining funds for the transaction pair, and the transaction mining funds can only be the currency of the current transaction pair, and other currencies are not allowed.
